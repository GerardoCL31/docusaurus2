# Despliegue

El frontend está preparado para hosting estático, por ejemplo en One.com.

El backend puede desplegarse en Render u otros servicios compatibles con Node.js y MongoDB.

La separación de despliegue mejora la flexibilidad y el mantenimiento del sistema.
